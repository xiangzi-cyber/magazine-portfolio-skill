# 素材采集表(Material Collection)

> 把"作品页要让招聘方相信的每一句话"反向拆成"需要什么样的素材来撑住"。
> 没有素材撑的主张 = 飘的话 = 删掉或改写。
>
> 这张表既是 **素材采集任务清单**,也是 **主张能不能站住的反向自检工具**。

---

## 怎么用(三步)

**Step 1 · 拆主张**
把作品页文案(hero、章节、数据条、CTA)里每一条让招聘方"必须相信"的话,提取成一行。

**Step 2 · 配佐证类型**
每条主张配 1-3 类佐证(下面五分法),写明"长什么样" —— 具体到"一张 GitHub 提交截图""一段微信对话截图""一份 PDF 缩略图"。

**Step 3 · 填"在哪 / 状态 / 备注"**
- **在哪**:写明素材所在平台(GitHub / 朋友圈 / 微信某群 / 电脑某文件夹 / 公众号 / 飞书 / 等)
- **状态**:
  - ✅ 已在手(电脑里能找到)
  - ⏳ 知道在哪、需要去取
  - ❌ 不在手且不确定能不能拿到
  - ⚠️ 涉及他人隐私,需评估是否能用
- **备注**:特殊处理(打码 / 裁剪 / 改写)、替代方案、放弃理由

---

## 五类佐证(每条主张选 1-3 类配上去)

| 类型 | 含义 | 典型形态 | 对招聘方的作用 |
|---|---|---|---|
| **过程截图(Process)** | 你做事过程中留下的痕迹 | 朋友圈进度截图、聊天对话、prompt 迭代版本、白板/笔记本照片、设计稿草图 | 证明"这是你真的做的,不是包装出来的" |
| **产物(Artifact)** | 最后做出来的东西本身 | PDF、设计源文件、代码仓库、视频、文档、智能体链接 | 证明"做出来的东西真的存在,可以打开看" |
| **第三方反馈(Voice)** | 别人对你做的事的反应 | 客户反馈截图、伙伴对话、引用、转发、评论 | 证明"不是你自夸,有人真在用 / 真说好" |
| **数据(Data)** | 可以量化的事实 | Git commit 数 / stars / 案例数 / 播放量 / 阅读量 / 交付天数 | 给"我说的"加一个客观刻度 |
| **时间锚点(Timestamp)** | 锁定时间的截图 | 朋友圈日期、GitHub 提交时间、文档修改时间、群聊时间戳 | 证明"我说的 30 天 / 12 小时是真的" |

> **关键判断**:同一条主张越多类型佐证叠加越硬;但**一条强佐证胜过三条弱佐证**。
> **隐私底线**:涉及他人姓名、组织、对话的素材必须先打码、得到对方同意,或改写成不可识别个体的描述。

## 跨项目素材采集源清单(把"散落在哪"集中起来)

> 用户原话:"素材散落在各个地方"。下表按**平台**反向归档,方便打开一个平台一次把里面所有素材都翻出来。

| 平台 / 来源 | 可能藏着的素材类型 | 当前状态 | 备注 |
|---|---|---|---|
| **GitHub** | commits / stars / contributors / README / 文档 / PR / issue / release | ⏳ | 项目相关仓库 |
| **微信朋友圈** | 进度截图、时间锚点、第三方点赞 / 评论 | ⏳ | 翻项目相关时间段 |
| **微信对话**(个人 / 群) | 第三方反馈、推力对话、用户原话 | ⚠️ 涉及隐私,需打码 | 关键合作老师 / 用户对话 |
| **公众号 / 播客** | 已发表内容、阅读数据、用户评论 | ⏳ | 列出已发文章清单 |
| **飞书 / Notion** | 工作文档、协作记录、版本历史 | ⏳ | 早期笔记 |
| **本地电脑文件夹** | 设计稿源文件 / 早期草稿 / PSD / 截图 | ⏳ | 建议归档到一个统一位置 |
| **各 AI 产品对话历史** | YouMind / Manus / Codex / 扣子 / Lovart / kimi 等 | ⏳ **过期风险** | **优先级最高**,产品历史会过期 |
| **kimi / 扣子智能体后台** | 智能体作品本身、使用数据 | ✅ 大部分还在 | 可截图 |
| **B 站 / 小红书 / 抖音** | 视频作品 / 播放数据 / 评论 | ⏳ | 看是否有可用作品 |

## 反向自检规则(关键)

**主张 - 佐证 闭环原则**:

> 某条主张如果**三类佐证都拿不到**,要么改主张、要么删主张 —— **坚决不写靠嘴说的东西**。

这是「服务视角原则」(见 SKILL.md Agent Behavior Specification §3 安全边界)的具体落地。

## 后续动作(填完表之后)

1. **样板对了之后**:Agent 用同一格式把另外 N 个项目的主张行也拉出来(预填 + 用户修)
2. **采集环节**:这张表填完后,变成"待办列表" —— 每条 ⏳ / ❌ 都对应一次具体动作(打开 GitHub 截图 / 翻朋友圈 / 翻微信 / 画一张图)
3. **优先级排序**:Agent 帮你按"先易后难 / 先必拿后可选"排序
4. **反向自检**:某条主张三类佐证都拿不到 → 要么改主张,要么删主张 —— 不写靠嘴说的东西
