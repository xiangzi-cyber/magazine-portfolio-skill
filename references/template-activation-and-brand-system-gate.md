# 模板激活与品牌系统门禁

> 这条门禁解决的是：**同一个 skill 被提到，但输出没有继承同一套品牌系统**。

## 什么时候触发

- 用户明确说“火种车那版生效了，但另一个项目没生效”
- 用户说“同一个 skill，为什么画风差这么多”
- 页面已经开始写，但视觉骨架像新搭了一套，而不是从母版延出来

## 先确认什么叫“生效”

品牌系统生效，不是看有没有写到 skill 名字，而是看下面四件事有没有同时成立：

1. **起步母版一致**：从 `assets/template.html` 或已验证的母版页出发
2. **类名家族一致**：仍在 `.hero` / `.act-cover` / `.section` / `.outro` 这套语法里
3. **字体与色彩 token 一致**：沿用 skill 的 token，不另起一套 serif / sans / 色彩系统
4. **节奏一致**：章节呼吸、版心、图片落位、证据节奏与母版相近

## 失败信号

- 页面主体类名变成 `.screen` / `.deck` / `.proof` / `.quote` 这种自建骨架
- 字体从轻量无衬线主导，切到另一套衬线主导系统
- 图片处理开始只顾“塞进去”，不再按角色分图墙 / 细节条 / 总览区
- 只是在旧页面上补局部样式，没有先证明母版已经接管

## 通过前检查

- [ ] 先读 `assets/template.html` 的 `<style>`，确认模板类存在
- [ ] 对照母版页，确认 hero / 章节封面 / 内容段 / 收尾的结构语法一致
- [ ] 确认没有把新项目写成另一套页面骨架
- [ ] 如果发现是“局部补救”而不是“母版继承”，先回退到模板起步

## 回退动作

如果没通过，不继续做文案润色或图片美化，先回到：

1. 模板起步
2. 类名预检
3. 母版结构对齐
4. 再填内容
