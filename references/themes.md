# 主题色预设(Themes)

6 套精心调配的主题色板,前 5 套从 [guizang-ppt-skill](https://github.com/op7418/guizang-ppt-skill) 直接继承,第 6 套从箱子主视觉取色,保证"电子杂志 × 电子墨水"美学不垮。

**不允许无依据自定义颜色** —— 没有主图/品牌源图时,只从以下预设中挑选;用户提供明确主图时,按"主色提取 -> token 发散 -> 组件落位"生成品牌色系,再沉淀为预设。

> **整站统一原则**:作品集所有页面用同一套主题色 ——
> 首页 / 关于页 / 项目页 1 / 项目页 2 / 项目页 N **不要换主题**。
> 整站颜色一致 = 视觉品牌记忆;颜色乱跳 = 招聘方认知摩擦。

---

## 使用方法

1. 基于求职方向推荐一套,或问用户选哪一套
2. 打开 `assets/template.html` 的 `<style>` 块
3. 找到开头的 `:root{` 块
4. **整体替换**标有"主题色"注释的 token 块。基础主题至少替换 6 行(`--ink` / `--ink-rgb` / `--paper` / `--paper-rgb` / `--paper-tint` / `--ink-tint`);第 6 套还要一并复制扩展品牌 token。
5. 其他 CSS 都走 `var(--...)`,无需任何其他改动

## 切换原则

- **整站只用一套主题** —— 作品集首页 / 关于页 / 所有项目页保持一致
- WebGL hero 背景的颜色由 `--ink` / `--paper` 自动适配,**不需要单独调整 shader**
- `currentColor` 驱动的 border / icon 会跟随 section 的 text color 自动适配
- `<title>` 文字和 `chrome` 元数据可以强化主题语义(例如牛皮纸配 "Vol.03 · 秋")

---

## ❌ 不要做的事

- ❌ **不允许混搭**(例如 ink 取墨水经典、paper 取沙丘)—— 彻底违和
- ❌ **不允许无依据自定义 hex 值** —— 没有主图时委婉拒绝并展示 6 套让选;有主图时必须先提取主色,再写成完整 token
- ❌ **不要逐处改 template.html 散落的颜色** —— 所有颜色走 var,改 :root 一处即可
- ❌ **不要在不同项目页用不同主题** —— 整站统一,加深品牌记忆

## 🟣 电紫 Explorer Violet (箱子代表紫)

**适合**:箱子探索者手记 / 个人品牌首页 / 主视觉头像驱动的作品集 / 方法沉淀 / 带有手写批注感的作品页。
**来源**:从主图 `6666.jpg` 的大面积背景紫提取,代表色集中在 `#6068F0` / `#6268F0`。
**调性**:荧光蓝紫作品牌核心,深墨蓝紫保证正文可读,金色/青蓝/粉色来自角色装备与面部细节,只做组件辅助。

```css
--ink:#15143A;
--ink-rgb:21,20,58;
--paper:#FAF8FF;
--paper-rgb:250,248,255;
--paper-tint:#EEF0FF;
--ink-tint:#4D55D9;
--brand-purple:#6268F0;
--brand-purple-rgb:98,104,240;
--line-purple:#4D55D9;
--accent-cyan:#1FB7F0;
--accent-gold:#E6B861;
--accent-rose:#D75C93;
--wash-violet:rgba(98,104,240,0.16);
--wash-cyan:rgba(31,183,240,0.10);
--wash-amber:rgba(230,184,97,0.14);
--annotation:#7A4E2A;
```

**品牌发散规则**:

- `--brand-purple` 是核心视觉,用于首页 Hero、关键数字、active 状态和最终收口。
- `--line-purple` 用于主线、分隔线、路径、边框和需要长期阅读的强调字。
- `--ink` 只负责正文可读性,不要拿荧光紫直接写长正文。
- `--accent-gold` / `--accent-cyan` / `--accent-rose` 由 `references/expedition-profiles.md` 注册为项目档案色;项目页只通过 `--accent-current` 渲染其中一种,不能喧宾夺主。
- `--wash-violet` 是背景承托,用于章节封面 / 引用块 / Hero;透明度宁可低,不要把整页刷成高饱和紫。
- `--annotation` 用于手写批注和旁注,保持批注棕,不进入蓝紫主线。
