# 规则门禁流程(Rule Gates)

本文件是规则单一事实源。每次使用本 skill 时,先识别任务阶段,只激活当前阶段需要的门禁;不要一次性把全部 checklist 当成工作记忆。

---

## 运行循环(Runtime Loop)

每次调用 skill 都按这个循环执行。这个循环比单条规则优先级更高。

```text
1. 识别任务类型 -> 2. 选择门禁 -> 3. 只读取对应规则包 -> 4. 执行阶段 -> 5. 阶段结束检查 -> 6. 继续 / 回退 / 停止
```

| 步骤 | 动作 | 产物 |
|---|---|---|
| 1. 识别任务类型 | 判断用户是在梳理、写作、进图、HTML、视觉修复还是验收 | 任务类型一句话 |
| 2. 选择门禁 | 从 G0-G7 中选 1-3 个本轮门禁 | `本轮激活门禁: [...]` |
| 3. 读取规则包 | 只读门禁指定文件,避免把全部规则塞进上下文 | 已读取文件列表 |
| 4. 执行阶段 | 按当前阶段产出文件 / 页面 / 清单 | 阶段产物 |
| 5. 阶段结束检查 | 按固定格式写检查结果 | `阶段结束检查` |
| 6. 继续 / 回退 / 停止 | 未通过则回退,通过才进入下一阶段 | 下一步决策 |

---

## 阶段结束检查协议

固定格式:

```text
阶段结束检查: [阶段名]
- 已产出: [文件 / 页面 / 清单]
- 已检查: [本阶段门禁编号]
- 未通过: [无 / 具体问题]
- 下一步: [继续到哪个阶段 / 回退补什么]
```

如果`未通过`不是“无”,下一步只能是回退补材料、降级主张、修图或重做验证,不能继续推进。

## P0 规则包

| 规则 ID | 名称 | 所属门禁 | 正文 | 失败回退 |
|---|---|---|---|---|
| P0-1 | 终稿口吻隔离 | G2/G5/G6 | `服务视角 / 招聘方视角` 只允许出现在 MD、内容架构稿、文案稿和检查清单中。最终 HTML 必须是成品作品页,不得出现“招聘方读到的是”“招聘方 get 到的是”“服务视角”“内部检查”或 `.service-note` / `.internal-note`。 | 回到 M7 转写,再生成 HTML |
| P0-2 | 文案冻结保护 | G4/G6 | 当用户已确认文案、当前任务只是"进图/加图/替换图"时,进入 Image-only Mode:只做入库命名、压缩处理、替换图片占位、验证加载、更新素材清单;不得新增标题、段落、解释性文案、方法论总结,不得重构页面。若需要做品牌总览图/证据板/复合图,必须另开任务。 | 撤回文案和结构改动 |
| P0-3 | 截图插入双模式 | G3/G4 | 当任务是插入产品/网页/工具/聊天/PPT/设计稿等截图时,默认做"截图证据板":不改截图内容,只在外部加入当前主题的背景、留边、细边框/窗口、轻阴影,并在图下放一句小字说明"这是什么/证明什么"。只有用户明确说"原封不动/保持原图/不要处理/裸放"时,才直接插入原图。 | 重新按指定模式处理截图 |
| P0-4 | 图片证据去重 | G4/G6 | 同一张图片默认只出现一次。同图有多重含义时,只放在最强语义位置;其他位置用不同证据、裁切、复合证据板或文字引用。确需复用,必须在图片落位表写明理由。 | 重新落位或补复用理由 |
| P0-5 | 模板激活保护 | G5/G7 | 同一个 skill 不是同一套效果;先证明页面确实继承了母版(template / 类名 / token / 节奏),再做局部优化。若页面骨架已经变成另一套(`.screen` / `.deck` / `.proof` 之类),先回到母版起步。 | 回到 `assets/template.html` 或已验证母版页 |
| P0-6 | 保义压缩 | G2/G7 | 保义压缩用于把长段落改成短判断:必须保留原意、事实、角色边界和结果证据,不能为了短而扭曲用户意思。 | 恢复原意后再压缩 |

## 门禁总表

| 门禁 | 触发时机 | 需要读取 | 必须产出 / 检查 | 不通过就停在 |
|---|---|---|---|---|
| G0 任务路由 | 收到需求第一步 | `SKILL.md` 路由表 | 判断是梳理、写作、进图、HTML、首页、视觉修复、整站范围外 | 需求澄清 |
| G1 源材料深读 | 素材多、个人经历强、用户说"假大空 / 没读懂 / 不像我" | `workflow.md` M3 | `原文摘录与真实问题拆解-XXX.md` | 不写文案、不做 HTML |
| G2 写作流程 | 写正文、文案稿、叙事重写、招聘方视角改写 | `workflow.md` M6-M7 | `作品页-XXX-文案稿.md` 和 `HTML内容架构稿`;每段回答原文依据、真实问题、解决动作、不写什么 | 不进 HTML |
| G3 图片入库 | 用户给压缩包、Eagle 图、截图、打印级大图 | `image-intake-and-screenshot-proof.md` | 原图不动、筛选、压缩、命名、落位表、缺口分级 | 不替换页面 |
| G4 图片冻结 | 用户说进图 / 加图 / 替换图,且文案已确认 | `image-intake-and-screenshot-proof.md` | 只改图片路径、alt、短 caption、素材清单 | 不改文案、不重构 |
| G5 HTML 生成 | 内容架构稿 / 文案稿 / 首页请求已通过,开始写 HTML | `sections.md` / `homepage.md` / `content-density.md` / `assets/template.html` / `template-activation-and-brand-system-gate.md` | 模板激活、类名预检、项目页 4 章节或首页 H01-H06、段落信息密度、主题色 | 不交付 HTML |
| G6 浏览器验收 | HTML 生成或视觉改动后 | `checklist.md` | broken image = 0、overflowX = 0、同类组件统一、图片语义对应 | 不最终交付 |
| G7 反馈迭代 | 用户说不舒服、空、乱、像模板、图文失衡 | `workflow.md` / `checklist.md` | 找同类问题,统一修;记录改动范围 | 不只修截图那一处 |

## 阶段结束检查表

| 阶段 | 结束时必须检查 | 对应门禁 / 文件 | 通过标准 | 没过怎么办 |
|---|---|---|---|---|
| M0 启动判断 | 项目是否已完成、素材状态、是否需要 7 问 | G0 / `workflow.md` | 路由清楚,没有把未完成项目当复盘写 | 回到需求澄清 |
| M1 7 问复盘 | Q1-Q7 是否答出关键事实,空白是否标记 | G1 / `7-questions.md` | 至少能回答项目、对象、真实痛点、展示物 | 继续问用户,不硬编 |
| M2 节点接力 | 节点是否归位到 Q3/Q4,用户原词是否保留 | G1 / `workflow.md` | 每个节点有归属和候选关键词 | 补问节点或标不确定 |
| M3 源材料深读 | 是否读到真实问题、动作、不完美、证据缺口 | G1 / `workflow.md` | 有`原文摘录与真实问题拆解-XXX.md` | 不写文案,继续读素材 |
| M4 主张-佐证 | 每条主张是否有 1-3 类证据 | G1 / `material-collection.md` | 无证据主张被删掉或降级 | 改主张 / 标待补 |
| M5 内容架构 | 4 章节 8 段是否一段一事,是否可合并 | G2 / `content-architecture.md` | 每段有主张、服务视角、视觉建议、素材状态 | 合并弱段 / 重排结构 |
| M6 文案稿 | 是否先事实段再标题金句 | G2 / `workflow.md` | 每段能答原文依据、真实问题、解决动作、不写什么 | 不进 HTML,重写文案稿 |
| M7 口吻隔离 | 是否只保留最终网页可见内容 | G2 / P0-1 | HTML 内容架构稿已去除内部审稿字段 | 回到文案稿转写 |
| V1 模板激活 | 模板与品牌系统是否继承 | G5 / P0-5 | 母版、class family、token、章节节奏都对得上 | 回到母版起步 |
| V3 图片填充 | 图片是否只改允许范围、是否去重 | G3/G4 / P0-2/P0-4 | 只改图,落位表更新,无未经说明的复用 | 撤回越权改动 |
| V5 浏览器验收 | 图片、横向溢出、同类组件、语义对应是否过关 | G6/G7 / `checklist.md` | broken image = 0, overflowX = 0, 图片证明所在段落 | 回到 HTML / 图片阶段修 |

## 任务类型到门禁映射

| 用户意图 / 关键词 | 激活门禁 | 第一读取文件 | 阶段结束检查重点 |
|---|---|---|---|
| 梳理项目 / 整理素材 / 捋项目 | G0 + G1 | `workflow.md` | 是否产出源材料深读或素材时间线 |
| 写文案 / 改叙事 / 招聘方视角 | G1 + G2 | `workflow.md` | 是否有文案稿,每段是否有事实依据 |
| 加图 / 进图 / 替换图 | G3 + G4 + G6 | `image-intake-and-screenshot-proof.md` | 是否只改图,是否更新落位表,是否验证加载 |
| 压缩包 / Eagle / 大图入库 | G3 + G6 | `image-intake-and-screenshot-proof.md` | 是否原图不动、命名压缩、缺口分级 |
| 生成 HTML / 重做页面 | G2 + G5 + G6 | `template-activation-and-brand-system-gate.md` + `checklist.md` | 是否有文案稿,模板是否激活,类名和章节是否过关 |
| 品牌系统没生效 / 同一个 skill 但画风不一致 | G5 + G7 | `template-activation-and-brand-system-gate.md` | 是否真的继承母版 class / token / 节奏,是否需要回到模板起步 |
| 视觉不舒服 / 空白大 / 图文乱 | G6 + G7 | `checklist.md` | 是否找同类问题统一修 |
| 最终交付 / 预览验收 | G6 | `checklist.md` | broken image / overflow / 图片语义是否过关 |

## 常见失效点

| 失效表现 | 原因 | 修正 |
|---|---|---|
| 写得很用力但空 | 跳过 G1,没有先读出真实问题 | 回到源材料深读 |
| 文案漂亮但不像用户 | 跳过 G2,先写标题 / 方法论 | 回到事实段 |
| 加图时页面被重写 | 跳过 G4 | 切回图片冻结任务 |
| 同一个 skill 但品牌系统没起来 | 跳过 G5 | 回到 template 或已验证母版页起步 |
| 规则太多记不住 | 一次加载全部 checklist | 每次只激活本阶段门禁 |
| 只修了一处截图问题 | 跳过 G7 | 找同类组件统一修 |
