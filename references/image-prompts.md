# 配图类型与提示词(Image Prompts)

作品集场景的图片需求**跟演讲 PPT 不同** —— 招聘方需要看到具体项目细节,所以**真实素材 > 生成图**。

> **优先级**:**真实素材 > 生成图**。
> 真实素材(项目交付物 / 过程截图 / 微信对话 / 朋友圈)比 AI 生成的"看着像"图,在招聘方眼里**力度差三倍**。
> 只有当真实素材**确实没有**或者**涉及隐私必须打码到无法识别**时,才考虑用 GPT-M 2.0 等工具补图。

## 作品集图片需求清单(按段落)

| 段 | 图类型 | 优先级 | 比例 | 说明 |
|---|---|---|---|---|
| Hero | 项目主视觉(成品) | ★★★ | 16:9 / 16:10 | 项目最终落地图 / LOGO / 主页截图 |
| 1B 用户分层 | 对照视觉(机械 vs 亲民等) | ★★ | 4:3 | 可用项目内素材 |
| 2A 流程压缩 | 流程对比图 | ★ | 16:10 | 信息图,可生成 |
| 2B 视觉分诊 | 批量产出 vs 原子图 | ★★★ | 1:1 / 4:3 | **必须真实**(过程截图 + 关键图) |
| 2C 两轮迭代 | (无图,Pipeline 代替) | — | — | — |
| 3A 工艺兜底 | (无图,数据卡代替) | — | — | — |
| 3B 衍生家族 | 衍生物成品 6 张 | ★★★ | 1:1(网格统一) | 母 LOGO + 字体 + 手举牌 + KT 板 + 玩偶 + 表情包 |
| 4A 方法沉淀 | (无图,纯文字) | — | — | — |

## 图片规格规范

### 命名规则

```
images/{段号}-{语义}.{ext}
```

例子:
- `01-hero.jpg`(Hero 主视觉)
- `1B-contrast.jpg`(用户分层对照)
- `2B-atom-grid.jpg`(批量产出墙)
- `2B-atom-final.jpg`(原子图单独放大)
- `3B-family-01-logo.png`(衍生家族第 1 个)
- `3B-family-02-font.png`
- ... 类推到 06

### 文件规格

- **单张 ≥ 1600px 宽**(避免大屏模糊)
- **JPG** → 照片 / 截图(质量 80-90)
- **PNG** → 透明 UI / 图标 / 信息图
- **总大小 < 8MB**(招聘方网络条件未知,首屏要快)
- **不嵌入文字标题** —— 文字交给 HTML 处理

### 比例硬规则

| 场景 | 比例 | 原因 |
|---|---|---|
| Hero 主视觉 | 16:9 | 大屏体面 |
| 左文右图 主图 | 4:3 / 16:10 | 跟正文比例协调 |
| 左文右图 方图 | 1:1 | 视觉锚 |
| 衍生家族 6 cell | 1:1 统一 | 不要混用 |
| 信息图 / 流程图 | 16:9 / 16:10 | 信息密度横向 |
| 竖图(可选) | 3:4 / 4:5 | 截图再设计 |

**不要用原图奇葩比例**(`2592/1798` 这种)。永远用标准比例。

## 真实素材分类

### A. 项目交付物(成品)

**最有力度** —— 招聘方眼里"做出来的东西"是硬证据。

- LOGO / 设计稿 / 海报 / KV
- 网站截图 / App 截图
- 印刷物照片(手举牌 / KT 板 / 名片)
- 实物照片(玩偶 / 周边)

**拍摄要求**:
- 真实照片优先(不要纯渲染)
- 平铺光 / 自然光
- 简洁背景(纯色或者无关背景)
- 单张构图,不要拼图

### B. 过程截图

**第二有力度** —— 证明你"真的做过"。

- AI 产品对话截图(YouMind / Manus / 扣子 / kimi)
- 设计稿迭代版本(v1 → v2 → 最终稿)
- Figma / Sketch 工作区截图
- GitHub commit 图 / contributions 热力图

**处理建议**:
- 涉及他人姓名 / 邮箱 / 头像 → 打码或裁切
- 公司 logo / 项目方信息 → 必须授权才用,否则改写为"某头部合作伙伴"等
- 截图统一比例(16:10 是最稳的)

### C. 第三方反馈

**最贵但最难拿** —— 别人说你的话,比你自己说强十倍。

- 客户 / 团队成员的对话截图(打码姓名)
- 朋友圈 / 公众号 评论 / 转发
- 用户反馈邮件(打码个人信息)

### D. 时间锚点

**项目真实性证据**。

- 朋友圈进度截图(带日期)
- GitHub 第一个 commit 时间戳
- 文件元数据时间戳
- 微信群通知时间(打码群名)

## 当真实素材不够时:GPT-M 2.0 配图

如果当前运行环境是 **Codex** 或类似有图像生成能力的环境,可以用 GPT-M 2.0 补 2-3 类生成图。

### 类型 1:对照视觉(机械 vs 亲民 / 旧 vs 新)

**用于**:1B 用户分层、2A 流程压缩。
**比例**:4:3 或 16:10。

**Prompt 模板**:
> 一张极简的对照视觉示意图,左半边是 [冷处理:机械感、齿轮、线框、灰色调],右半边是 [暖处理:温暖感、卡通、圆润、饱和色调]。整体风格:杂志墨水风、电子杂志风、克制留白。比例 4:3。**不要在图里加任何文字**(文字由 HTML 处理)。

### 类型 2:信息图 / 流程图 / 系统关系图

**用于**:2A 流程压缩(可补)、复杂方法论说明。
**比例**:16:9 或 16:10。

**Prompt 模板**:
> 一张极简的[流程图 / 系统图 / 对比图],描绘 [具体流程:接手 → 文字方案 → 审议 → 修改 → 出图]。风格:杂志墨水风、深色文字 + 米色背景、衬线字体感、克制留白。比例 16:10。**图里的所有文字必须是中文**(如果作品集语言是中文)。

### 类型 3:截图再设计

**用于**:2B 视觉分诊(批量草图墙)、过程截图统一化。
**比例**:1:1 或 16:10。

**Prompt 模板**:
> 把多张产品截图(已上传)拼贴成一张统一比例的墙,网格 4×3。每张截图保留核心 UI,周围用米色填充统一比例。整体风格:杂志墨水风、克制留白、中性色调。**不要加边框 / 阴影 / 标题**。比例 1:1。

## 生成图的硬规则

1. **不要让图自带文字标题**(标题 / 页脚 / 角标)—— 文字归 HTML
2. **比例必须匹配落位**(参见上面的比例表)
3. **保留核心视觉,周围填充统一**(不要让裁切区域占主导)
4. **风格必须贴合"电子杂志 × 电子墨水"基调**(墨水流动 / 杂志网格 / 克制留白)
5. **生成后再用 PS 微调**(锐化 / 色调 / 裁切),不要直接用 raw 输出

## 一些已知陷阱

### 陷阱 1:用 AI 生成所有图

**问题**:招聘方一眼看出"全是 AI 生成的"——失去真实感。
**解决**:**70% 真实素材 + 30% 生成图** 是健康比例。

### 陷阱 2:截图比例混乱

**问题**:衍生家族 6 cell 一个 1:1 一个 4:3,网格塌。
**解决**:**同组必须同比例**。

### 陷阱 3:涉及隐私不打码

**问题**:微信对话出现真实姓名 / 头像 / 公司 logo,违反隐私。
**解决**:**打码姓名 + 头像 + 公司 logo + 项目敏感名**;或改写成"伙伴 A"之类。

### 陷阱 4:图像分辨率太低

**问题**:1600px 以下在大屏模糊。
**解决**:**图源 ≥ 1600px**,实在不够就用低饱和度处理掩盖模糊感。

### 陷阱 5:把 PPT 当作品集图

**问题**:把演讲 PPT 截图直接放作品集 —— PPT 有页脚 / 页码 / 大标题,在作品集里是冗余。
**解决**:**重新裁切**或**重做 mockup**。
